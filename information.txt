# HRMS (Human Resource Management System) - Module Documentation

## Overview
This is a full-stack HRMS application built with Django REST Framework (backend) and React (frontend) using Vite as the build tool.

---

## BACKEND MODULES (Django)

### Core Django Framework
- **Django==5.2.8**
  - Function: Main web framework providing MVC architecture, ORM, authentication, admin interface
  - How it works: Handles HTTP requests, database operations, user authentication, and serves REST API endpoints
  - Key features: Built-in admin panel, ORM, security features, URL routing

### Database & API
- **djangorestframework==3.16.1**
  - Function: Provides powerful and flexible toolkit for building Web APIs
  - How it works: Extends Django to create RESTful APIs with serializers, viewsets, and authentication
  - Key features: API serialization, permission classes, viewsets, pagination

### CORS Support
- **django-cors-headers==4.9.0**
  - Function: Enables Cross-Origin Resource Sharing (CORS) for Django APIs
  - How it works: Adds CORS headers to API responses allowing frontend on different port to communicate
  - Key features: Configurable allowed origins, credentials support

### Database Components
- **sqlparse==0.5.3**
  - Function: SQL parsing and formatting library
  - How it works: Used by Django for SQL query parsing and formatting
  - Key features: SQL validation, formatting, parsing

### ASGI Support
- **asgiref==3.11.0**
  - Function: ASGI reference implementation and utilities
  - How it works: Provides ASGI (Asynchronous Server Gateway Interface) support for Django
  - Key features: Async request handling, middleware support

### Django Allauth (Authentication)
- **django-allauth** (Not in requirements.txt but referenced in settings)
  - Function: Comprehensive authentication system with social login support
  - How it works: Handles user registration, login, logout, email verification, and social authentication
  - Key features: Social login (Google), email verification, account management

## PROJECT STRUCTURE & DATA FLOW

### Backend (Django)
```
hrms_backend/
├── hrms_backend/          # Project configuration
│   ├── settings.py        # Django settings including CORS, auth, database
│   ├── urls.py           # Main URL routing
│   └── wsgi.py           # WSGI configuration
└── api/                  # Main Django app
    ├── models.py         # Database models (Todo, User relationships)
    ├── views.py          # API endpoints and business logic
    ├── serializers.py    # Data serialization for API
    ├── urls.py          # API URL routing
    └── admin.py         # Django admin configuration
```

### Frontend (React)
```
frontend/src/
├── components/           # React components
│   ├── App.jsx          # Main application with routing
│   ├── AdminDashboard.jsx # Admin role selection interface
│   ├── HODDashboard.jsx  # HOD-specific dashboard
│   ├── EmployeeDashboard.jsx # Employee dashboard
│   └── [Other components...] # Feature-specific components
├── App.css              # Application styles
└── main.jsx            # Application entry point
```

### Data Flow
1. **Frontend** makes HTTP requests using **axios** to Django backend
2. **Django** receives requests via **djangorestframework** endpoints
3. **Django ORM** interacts with **SQLite** database
4. **CORS headers** (django-cors-headers) allow cross-origin requests
5. **React Router** handles client-side navigation
6. **Vite** serves frontend with proxy to backend API

### Authentication Flow
1. User logs in via React form
2. Credentials sent to Django backend
3. Django authenticates using built-in auth system
4. JWT/session tokens returned to frontend
5. Frontend stores tokens and includes in subsequent requests
6. Protected routes check authentication status

### Development Workflow
1. **Vite dev server** runs on port 5173
2. **Django development server** runs on port 8000
3. Vite proxy forwards `/api/*` requests to Django backend
4. **Hot Module Replacement** provides instant frontend updates
5. **ESLint** maintains code quality during development

## DEPLOYMENT CONSIDERATIONS

### Production Requirements
1. Replace SQLite with PostgreSQL/MySQL for production
2. Configure proper CORS origins for production domain
3. Set up proper authentication (JWT/OAuth)
4. Configure HTTPS and security headers
5. Set up proper logging and monitoring
6. Configure static file serving (CDN recommended)

### Security Notes
- Django secret key should be environment variable in production
- CORS should be restricted to specific domains
- Database credentials should use environment variables
- Enable Django security middleware in production
- Implement rate limiting for API endpoints

## API ENDPOINTS SUMMARY

### Authentication
- POST /api/login/ - User login
- POST /api/logout/ - User logout

### Dashboard Data
- GET /api/dashboard/ - General dashboard statistics
- GET /api/hod/ - HOD-specific data
- GET /api/principal/ - Principal-specific data
- GET /api/employees/ - Employee data

### Management
- GET/POST /api/leave-request/ - Leave management
- GET/POST /api/attendance/ - Attendance management
- GET/POST /api/announcements/ - Announcement management
- GET/POST /api/departments/ - Department management

### File Operations
- File upload/download support for employee documents
- Profile picture management
- Document storage with Django's file system
